Kaggle Housing Price Prediction with Linear Regression

February 9, 2026 | 6 min read

I wanted to work on a Kaggle dataset/competition. I have worked on and used logistic regression for the Titanic dataset, so I wanted to try my hand at another predictive problem: the House Prices competition. The main goal here was to use the 79 attributes provided to predict the sale price of a house. Since this is a regression problem (predicting a continuous value rather than a category), I used linear regression and wanted to see how accurate it could be.

Data Cleaning

First, I cleaned the data by removing columns with a large number of null values, and then imputing the remaining columns with the mean (for numeric columns) and mode (for categorical columns). I then used a one-hot encoder to convert categorical variables into a binary format.

One-Hot Encoding

Here is the code for the encoder:

# Create and fit encoder on training data
encoder = OneHotEncoder(sparse_output=False, drop='first', handle_unknown='ignore')
encoder.fit(X_train_filled[object_cols])

# Transform both train and test
train_encoded = encoder.transform(X_train_filled[object_cols])
test_encoded = encoder.transform(X_test_filled[object_cols])

# Convert to DataFrames
train_encoded_df = pd.DataFrame(
    train_encoded,
    columns=encoder.get_feature_names_out(object_cols),
    index=X_train_filled.index
)

test_encoded_df = pd.DataFrame(
    test_encoded,
    columns=encoder.get_feature_names_out(object_cols),
    index=X_test_filled.index
)

# Combine with numeric columns
X_train_encoded = pd.concat([X_train_filled[numeric_cols], train_encoded_df], axis=1)
X_test_encoded = pd.concat([X_test_filled[numeric_cols], test_encoded_df], axis=1)

print(f"Train shape: {X_train_encoded.shape}")
print(f"Test shape: {X_test_encoded.shape}")

This was built with the scikit-learn library in Python, which provides a wide range of simple and efficient tools for data analysis and predictive modeling, built upon the Python scientific computing stack (NumPy, SciPy, and Matplotlib).

Training the Model

I used the linear regression model built into scikit-learn. Here is the code for that:

model = LinearRegression()
model.fit(X_train_encoded, y_train)

# Make predictions on training set
y_train_pred = model.predict(X_train_encoded)

# Evaluate on training data
print("Training R² Score:", r2_score(y_train, y_train_pred))
print("Training RMSE:", np.sqrt(mean_squared_error(y_train, y_train_pred)))
print("Training MAE:", mean_absolute_error(y_train, y_train_pred))

# Make predictions on test set
y_test_pred = model.predict(X_test_encoded)

Results

After data preprocessing and training the model, here were my training scores:

Training R² Score: 0.9301102689204063
Training RMSE: 20994.755492882352
Training MAE: 13511.797268099297

R² Score — 0.93

The R² score describes how much of the variance in the target variable is captured by our model. A score of 0.93 means the model explains about 93% of the variance in housing prices using the features in this dataset.

RMSE (Root Mean Squared Error) — $20,995

The RMSE is the square root of the average squared prediction error. Because it squares the errors before averaging, it is more sensitive to large errors and can be heavily affected by outliers.

MAE (Mean Absolute Error) — $13,512

The MAE is the average of the absolute prediction errors. It gives a more typical sense of the average error. This means the model's predictions are typically about $13,500 above or below the actual price.

Conclusion

This is an okay first attempt at learning and utilizing linear regression to make a prediction. Some ways I have been reading up on to improve my model include:

  • Different models: Random Forest or Gradient Boosting (XGBoost, LightGBM)
  • Feature engineering: Reducing dimensionality while also creating more meaningful variables
  • Handle outliers: Identifying and addressing extreme values that skew the model
  • Feature selection: Picking the most informative features
  • Better missing value handling: More sophisticated imputation strategies
  • Cross-validation: More robust evaluation of model performance
  • Scaling/Normalization: Standardizing feature ranges
  • Ensemble methods: Combining multiple models for better predictions
  • Target transformation: Applying log transforms to the target variable

I will start with feature engineering and will cover that in a new blog post because I'm sure there is a lot to learn there.
